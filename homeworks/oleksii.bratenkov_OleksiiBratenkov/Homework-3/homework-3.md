# ООП в программировании #

## Клас ##
Клас - самолет.

## Объект ##
- Пассажирский самолет.


##Полиморфизм##
Для примера возьмем метод "взлет". У пассажирского самолета и истребителя этот метод будет иметь разную реализацию.

##Наследование##
Самолет-топливозаправщик - наследник пассажирского самолета.

## Инкапсуляция ##
Пилоту, который управляет пассажирским самолет не обязательно знать - на какой скорости проходит воздух через лопатки турбокомпрессора. 

# Принципы SOLID #

- single responsibility principle. Пассажирский самолет только должен только перевозить людей. 
- Open-closed. Если поставить внутри салона дополнительные сидения - это будет расширение, но не изменение.
- Liskov substitution. Для полета можно использовать не только пассажирский, но и любой другой тип самолетов.
- Interface segregation. Управление пассажирским самолетом и его обслуживание имеют разные интерфейсы.
- Dependency inversion. При стоянке - пассажирский самолет не должен зависеть от функционирования конкретного прибора в кабине, но от работы ВСУ (вспомогательной силовой установки, которая генерирует ток при стоянке на земле) должен поскольку ВСУ генерирует ток и для конкретного прибора и без этого, последний не будет работать. Поэтому, самолет должен зависеть от ВСУ.
